# SECURITY ポリシー

このプロジェクトは確定申告のための個人財務情報・各種サービスの認証情報・ブラウザセッションを扱うため、以下のセキュリティポリシーを遵守すること。

## 1. 機密ファイルと取扱区分

| パス | 機密度 | Claude Code (生成 AI) の取扱 |
|------|------|----------|
| `.env` | 高（認証情報） | **読み書き禁止**（`.claude/settings.json` で deny） |
| `.env.local` 等 | 高 | **読み書き禁止** |
| `.playwright-profile/` | 高（ブラウザセッション・ログイントークン） | 読み取りはできるが触れない方針。git 管理外 |
| `invoices/` | 中（個人の取引額・取引先） | タスク上必要な範囲で読む（PDF 解析等）。git 管理外 |
| `.playwright-mcp/` | 中（一時 DL ファイル・ログ） | 読み書き可だが内容は機密扱い。git 管理外 |

## 2. `.env` の運用ルール

現状 `.env` は未使用（MCP は OAuth／認証情報不要）。将来必要になった場合に備えて deny ルールと本ルールは維持する。

- **値の追加・変更はユーザー本人が `.env` を直接編集**する。Claude Code は `.env` に触れない
- 生成 AI が新しい環境変数を必要とした場合：
  1. **生成 AI はキー名のみをユーザーに伝える**（例: 「`<KEY>` を `.env` に追加してください」。値は見せない・生成しない）
  2. ユーザーが手動で `.env` を作成・更新する
- 既存の値が変更された旨を生成 AI に伝えたい場合も、生成 AI に値を見せず「変更しました」とだけ伝える

## 3. 強制設定（`.claude/settings.json`）

以下のツールアクセスを deny にしている：

- ファイルツール：`Read/Edit/Write` の `./.env`、`./.env.local`
- Bash：`.env` を対象とする主要コマンド（`cat`、`less`、`head`、`tail`、`source`、`.`（dot）、`grep`、`sed`、`awk`、`cp`、`mv`、`od`、`xxd`、`hexdump`）

**注意**：Bash は無数の読み取り方法があり deny で完全には防げない。`base64 .env`、`python -c "open('.env').read()"`、`curl --data-binary @.env` などは個別に防いでいない。生成 AI 自身が本ポリシーに従って `.env` を読まないことが大前提。

## 4. MCP サーバーの認証情報

`.mcp.json` は git にコミットされるため、認証情報を直書きしない。`.env` の環境変数を `env` フィールドで参照する形式に統一：

NG:
```json
{
  "mcpServers": {
    "example": {
      "command": "npx",
      "args": ["-y", "example-mcp", "--token", "actual_secret_value"]
    }
  }
}
```

OK:
```json
{
  "mcpServers": {
    "example": {
      "command": "npx",
      "args": ["-y", "example-mcp"],
      "env": {
        "EXAMPLE_TOKEN": "${EXAMPLE_TOKEN}"
      }
    }
  }
}
```

新規 MCP 追加時のチェック：
- `command` と `args` が信頼できるソース（公式パッケージ等）か
- どの環境変数・ファイルにアクセスするか
- 外部に通信する場合、送信先と内容

## 5. git 操作の注意

- **コミット禁止**：`.env`、`.env.local`、`invoices/**`、`.playwright-profile/**`、`.playwright-mcp/**`
- `git add .` や `git add -A` を避け、追加ファイルは明示的にパス指定する
- リポジトリ Visibility は **Private を維持**（誤って公開しないよう注意）
- 万一公開リポジトリに認証情報を含むコミットをした場合、`git reset` だけでは履歴に残るため、対応サービスのトークンを **直ちに再発行**

## 6. 出力時の注意

- 認証情報・トークン・セッションクッキーを Claude の出力（メッセージ、コミットメッセージ、PR 本文、Slack 等）に含めない
- スクリーンショット取得時は機密情報が画面に映っていないか確認
- ログファイル（`.playwright-mcp/` 配下のコンソールログ含む）は git 管理外に置く

## 7. サードパーティ依存

- npm パッケージは `@latest` で起動するものがある（例：`@playwright/mcp@latest`）→ サプライチェーン攻撃のリスクは把握する
- 重要：信頼できないリポジトリからクローンした MCP サーバー定義 (`.mcp.json`) を安易に承認しない
- マシン共有の場合、Playwright のユーザーデータディレクトリ (`.playwright-profile/`) を他人がアクセスできない権限に設定する

## 8. インシデント対応

漏洩の疑いがある場合の手順：

1. **対応する各サービスのトークン・パスワードを直ちに再発行**
2. ブラウザセッション (`.playwright-profile/`) を破棄して再ログイン
3. 漏洩経路を特定（git 履歴・出力ログ・スクリーンショット等）
4. 必要に応じて関係者・サービス提供元に通知
5. ポリシー文書（本ファイル）と `.claude/settings.json` を見直し、再発防止策を追加

## 9. 個人情報を Git 追跡ファイルに書かないルール（重要）

`CLAUDE.md`・`.claude/config/services/*.yml`・`.claude/skills/**/SKILL.md`・`docs/`（＝ Git 追跡される全ファイル）には
**個人情報を一切書かない**。実値は **Git 管理外の `local.yml` に置き**、追跡ファイル側は
**プレースホルダー**（`カードA補助科目`・`example.com` 等）または **`local.yml` 参照**で表現する。

**個人情報に該当するもの（書いてはいけない）：**

- 取引先コード（`A00000000X`）、事業者番号、事業者名・屋号、氏名
- カード/銀行の補助科目名、カード会社名、銀行名
- 所有ドメイン名、メールアドレス
- 請求書番号・レシート番号、**実際の金額**、仕訳No.

**書いてよいもの（PII ではない）：** 会計分類（通信費・未払金等）、税区分の方針、公開ベンダー名（Anthropic 等）、
MF 明細キーワード（`CLAUDE.AI` 等）、サービス名・スキル名。

**特に注意するタイミング：** 新サービス追加時、スキルの「確認済み実例」記録時、初回実行で得た実値を反映する時。
これらの作業後は、追跡ファイルを対象に個人情報トークンを grep し、混入ゼロを確認してからコミットすること
（実値は `local.yml`／参照記述のみが残る状態が正）。経緯は Phase 13（PII マスキング）参照。
