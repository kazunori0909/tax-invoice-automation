# 個人設定（local.yml）セットアップガイド

`.claude/config/local.yml` は、あなたの契約内容（利用サービス・決済カード等）を書く個人設定ファイルです。  
**Git 管理外**なので、カード名やドメイン名などの実値をそのまま書いて問題ありません。

サービス共通の仕様（検索キーワード・摘要テンプレ等）は隣の `services/*.yml`（Git 管理）にあり、
ユーザーが編集するのは原則 `local.yml` だけです。

## 初回セットアップ
`local.example.yml` をコピーし`local.yml`にリネームします。  
コピーした `local.yml` を、このガイドを見ながら自分の契約に合わせて編集します。


## 記述サンプル
### 請求書ダウンロードスキルのみ使用する場合
`services:` にサービスキーを列挙（＋wix の `sites` / moomoo の `domains`）する
```yaml
# キーを列挙したサービスだけが処理対象になる。使わないサービスはキーごと削除する。
services:
  moneyforward:
    frequency: yearly    # 年額契約の場合のみ記載
  anthropic:             # キーがあればダウンロード対象
  google-ai:
  xserver:
    frequency: yearly
    billing_month: 4     # 年次更新の請求月（省略可）
  wix:                   # 管理しているWixサイト全ての請求書をダウンロードする

# 仕訳登録（/register・/credit-payment）を使う場合のみ必要。ダウンロードだけなら削除してOK。
# credit_cards:
```

### 請求書ダウンロードと仕訳登録スキルを使用する場合
上記の請求書ダウンロードの項目に加えて、各サービスの `card:` と `credit_cards:` セクションを記載する
```yaml
# キーを列挙したサービスだけが処理対象になる。使わないサービスはキーごと削除する。
services:
  moneyforward:
    frequency: yearly
    card: ⚪︎⚪︎カード       # データ連携されているカード情報
  anthropic:
    card: ⚪︎⚪︎カード       # データ連携されているカード情報
  google-ai:
    card: ⚪︎⚪︎カード       # データ連携されているカード情報
  xserver:
    frequency: yearly
    billing_month: 4
    card: △△カード        # データ連携されているカード情報
  wix:                   
    card: ⚪︎⚪︎カード

# 仕訳登録（/register・/credit-payment）を使う場合のみ必要。ダウンロードだけなら削除してOK。
credit_cards:
  - name: ⚪︎⚪︎カード
    type: business
    closing: month_end            # 月末締め
    payment_day: 26               # 翌月の支払日
  - name: △△カード
    type: personal
```

## `services:` の書き方
`services:` の下に**キーを書いたサービスだけが処理対象**になります。  
使わないサービスはキーごと削除（またはコメントアウト）してください。

```yaml
services:
  anthropic:             # キーを書く＝利用中の宣言。設定が無くてもキーだけで OK
  # google-ai:           # キーがコメントアウトされているため、ダウンロード対象外
  xserver:
    frequency: yearly    # 年額契約の場合のみ
    billing_month: 4     # 年次更新の請求月
    card: ○○カード        # 仕訳登録も使う場合のみ
```

### 仕訳登録用 共通設定

| キー | 意味 | いつ書く |
|---|---|---|
| `frequency` | 請求サイクル<br/>（`yearly` で年額に切替） | 年額契約の場合のみ（省略時は月額） |
| `billing_month` | 年次更新の請求月<br/>（1〜12） | `frequency: yearly` の場合。<br/>省略しても `/register` が過去の仕訳から学習する |
| `card` | 決済カードの MF 連携上の表示名 | 仕訳登録も使う場合。<br/>`credit_cards:` の `name` と同じ文字列にする |


### wix用 設定 （`sites` の書き方)
| キー | 意味 | いつ書く |
|---|---|---|
| `sites` | サイトの一覧（`- name: example.com`） | 省略時は支払い履歴に表示される全サイトが対象 |

```yaml
  wix:
    sites:
      - name: example.com  # 複数サイトを管理している場合、個別の請求書が欲しい場合記載
```

### moomoo用 設定 (`domains` の書き方)
| キー | 意味 | いつ書く |
|---|---|---|
| `domains` | ドメインの一覧 | moomoo では必須 |

```yaml
  moomoo:
    domains:
      - fqdn: example.com     # フルのドメイン名。ファイル名（`.`→`_` 置換）・仕訳の摘要に使う
        billing_month: 2      # 年次更新の請求月
```

## `credit_cards:` の書き方
決済カードの登録（仕訳登録を使う場合のみ）必要な項目です。  
クレジットカードの利用に関しては、以下のパターンがあると想定しています。
- 事業専用クレジットカード
- 副業等のプライベート、個人利用クレジットカード

貸方は**MF 上でそのカードの連携口座がどの勘定科目配下に登録されているか**で決まります。  
事業用は「未払金」配下、個人利用は「事業主借」配下に
連携口座を登録してください（MF のデータ連携の口座設定画面で行う操作です）。  

個人利用カードなのに貸方が「未払金」で入ってくる場合は、連携口座の設定を事業主借配下へ変更してください  
（変更後の課金分から反映されます。過去の仕訳は自動では直りません）。

``` yaml
- name: ⚪︎⚪︎カード
  type: business
  closing: month_end            # 月末締め
  payment_day: 26               # 翌月の支払日

- name: △△カード
  type: personal
```


- `name` は `services.<svc>.card` と**同じ文字列**にしてください（名前で紐づきます）。  
  MF ホーム画面の「未仕訳 N 件」に表示される名称をそのまま使うのが確実です。
- 事業用カード（`type: business`）の `closing`・`payment_day`
  は `/credit-payment` が引き落とし予定日を計算するために使います。
- 個人利用カード（`type: personal`）は補助科目名を書く必要はありません。連携口座の設定で
  決まる値がそのまま使われます。

## `cache:` について
スキルの自動生成領域であり、**手動で設定する必要はありません。**  

`cache:` セクションはスキルが実行時に自動で書き込みます
（取引先コード・月額の請求日アンカー・年額の請求月）。  
動作がおかしいときは、該当サービスの行（または `cache:` セクションごと）を削除すれば次回実行時に再生成されます。

## 書かなくてよいもの

- **借方の勘定科目・補助科目**
  - MF の自動仕訳ルールに委譲します。初回の `/register` で
  確認しながら設定すれば、以後は MF が自動で補完します
- **貸方（事業主借・未払金）の補助科目名**
  - 連携口座の設定で決まる値をそのまま使うため、`credit_cards:` に書く必要はありません
- **勘定科目 ID・取引先コード**
  - スキルが MF から自動取得します
- **認証情報（ID・パスワード等）**
  - 一切書きません。ログインはユーザーがブラウザで行います。
