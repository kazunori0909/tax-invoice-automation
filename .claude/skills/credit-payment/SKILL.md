# クレジットカード引き落とし仕訳 確認・登録 Skill

## 概要

**未払金型（`type: business`）カード**の引き落とし日に対応する仕訳を確認・登録する。
個人用カード（`type: personal`）は課金時に事業主借で直接記帳される（register 管理）ため**対象外**。

| タイミング | 借方 | 貸方 |
|---|---|---|
| サービス課金時（連携・自動） | 通信費 | 未払金（カード補助科目） |
| **カード引き落とし時（本スキル対象）** | **未払金（カード補助科目）** | **普通預金（引き落とし口座）** |

引き落とし口座も MF 連携済みなら引き落とし仕訳は MF が自動生成されるため、本スキルは
その存在・金額の**確認**が主目的。未連携の場合のみ登録する。

## 前提条件

- MoneyForward MCP が認証済み
- `.claude/config/local.yml` が存在する（`credit_cards:` セクションを含む）

## 引数

対象月数（デフォルト: 直近3ヶ月）。ユーザーから「2026年5月分だけ」「直近6ヶ月」などの指定があればそれに従う。

---

## Step 0: カード設定の読み込みと勘定科目IDの解決

### 0.1 カード設定の読み込み

`.claude/config/local.yml` を読み込み、`credit_cards:` セクションからチェック対象カードの一覧を取得する。

```
Read: .claude/config/local.yml
```

**対象カードの絞り込み**：`type: personal`（事業主借直接）のカードは**対象外**（register が課金時に事業主借で登録済み）。`type: business`（または `type` 省略）のカードのみ処理する。business カードが 0 件なら「対象カードなし（個人用カードは register 管理）」とレポートして終了する。

### 0.2 勘定科目・補助科目IDの解決

勘定科目は `type: business` から導出する（config に書かない）：**借方＝未払金／貸方＝普通預金**。
`mfc_ca_getAccounts` で `未払金`・`普通預金` の ID を解決する。

補助科目（未払金＝カード／普通預金＝引き落とし口座）は **config 指定は任意**：

- **引き落とし口座も連携済み**（＝引き落とし仕訳を MF が自動生成）：その自動生成仕訳から
  借方（未払金）・貸方（普通預金）の補助科目 ID を**動的に取得**して確認に使う（config 記述不要）。
- **未連携で手動登録する場合**：`card_sub_account`・`bank_sub_account` を config から読み
  `mfc_ca_getSubAccounts` で ID 解決する。未指定なら手動登録の前にユーザーに確認する。

| 変数名 | 取得元 | 対応する名前 |
|---|---|---|
| `card_account_id` | `mfc_ca_getAccounts` | `未払金`（type から固定） |
| `bank_account_id` | `mfc_ca_getAccounts` | `普通預金`（type から固定） |
| `card_sub_account_id` | 自動仕訳から取得／`getSubAccounts`（`card_sub_account`） | 任意 |
| `bank_sub_account_id` | 自動仕訳から取得／`getSubAccounts`（`bank_sub_account`） | 任意 |

勘定科目（未払金・普通預金）が見つからない場合はユーザーにエラーを報告して処理を中断する。

---

## Step 1: チェック対象期間の算出

今日の日付を基準に、直近 N ヶ月分の引き落とし日と対応する締め期間を計算する。

**カードA（月末締め・翌月26日払い）の例（本日: 2026-05-24）:**

| 引き落とし日 | 締め期間（費用発生期間） |
|---|---|
| 2026-03-26 | 2026-02-01 〜 2026-02-28 |
| 2026-04-26 | 2026-03-01 〜 2026-03-31 |
| 2026-05-26 | 2026-04-01 〜 2026-04-30 |

**算出ロジック:**
- `closing: month_end` の場合: 締め期間 = 引き落とし月 -1 ヶ月の 1日〜末日
- `payment_day: 26` の場合: 引き落とし日 = 各月の26日

**営業日調整（重要）:**  
指定日が土曜・日曜・日本の祝日の場合は翌営業日に繰り越す。  
例: 2026-04-26（日曜）→ 2026-04-27（月曜）  
祝日カレンダーを参照し、連休の場合は連休明けの翌営業日まで繰り越す。

**締め完了フラグの判定:**  
各締め期間について `締め期間の末日 < 今日` であれば「締め完了」とみなす。  
例: 今日 2026-05-24 の場合、4月分（末日 4/30）は締め完了、5月分（末日 5/31）は締め未完了。

---

## Step 2: 費用仕訳の集計（期待引き落とし額の算出）

各締め期間について、対象カードの費用仕訳を MF から取得・合計する。

```
mfc_ca_getJournals
  from: YYYY-MM-01（締め期間の開始日）
  to:   YYYY-MM-DD（締め期間の末日）
```

取得した仕訳から以下の条件に合致するものを抽出し合計する:
- 貸方勘定科目 = 未払金（`card_account_id` に対応）
- 貸方補助科目 = カードの補助科目（`card_sub_account_id` に対応）

この合計金額 = **期待引き落とし額**

期待引き落とし額が 0 円の場合は、対象期間中に該当カードでの支出がないため **スキップ** する。

---

## Step 3: 引き落とし仕訳の存在確認

各引き落とし日について、既存の仕訳を確認する。

```
mfc_ca_getJournals
  from: YYYY-MM-26（引き落とし日）
  to:   YYYY-MM-26（引き落とし日）
```

取得した仕訳から以下の条件で抽出する:
- 借方勘定科目 = 未払金（`card_account_id`）
- 借方補助科目 = カード補助科目（`card_sub_account_id`）
- 貸方勘定科目 = 引き落とし口座（`bank_account_id`。business は普通預金）
- 貸方補助科目 = 引き落とし口座の補助科目（`bank_sub_account_id`）

---

## Step 4: 結果の分類とレポート表示

各月を以下の4パターンに分類する。

| パターン | 条件 | 次のアクション |
|---|---|---|
| ✅ 正常 | 仕訳あり・金額一致 | スキップ |
| 🚀 自動登録 | 仕訳なし・**締め完了**（締め期間末日 < 今日） | 確認なしで即登録（Step 5.1） |
| ⏳ 締め未完了 | 仕訳なし・締め期間がまだ終わっていない | スキップ（締め後に再実行） |
| ⚠️ 金額不一致 | 仕訳あり・金額不一致 | **ユーザー確認後**に修正（Step 5.2） |

**レポート表示例:**

```
引き落とし仕訳 確認レポート（カードA名称）:

  ✅ 2026-02-26: 登録済み・金額一致（¥XX,XXX）
  🚀 2026-03-26: 未登録（締め完了）→ 自動登録します（期待額 ¥X,XXX）
  ⚠️ 2026-04-27: 金額不一致（登録済み ¥X,XXX ≠ 期待額 ¥Y,YYY）→ 修正しますか？
  ⏳ 2026-06-26: 締め未完了（5月分、締め日 5/31 未到来）→ スキップ
```

- 🚀 自動登録はユーザー確認なしで即実行する
- ⚠️ 金額不一致のみユーザー確認を求める（修正件数が 0 件なら確認不要で終了）
- ⏳ 締め未完了はスキップし、レポートにのみ記載する

---

## Step 5: 仕訳の登録・修正

### 5.1 未登録の仕訳を自動登録（締め完了分）

```
mfc_ca_postJournals
  journals:
    - debit_account_id:    <card_account_id>
      debit_sub_account_id: <card_sub_account_id>
      credit_account_id:   <bank_account_id>
      credit_sub_account_id: <bank_sub_account_id>
      amount:   <期待引き落とし額>
      date:     YYYY-MM-26
      memo:     "<name> 引き落とし（YYYY年M月分）"
```

**注意:** 免税事業者のため `invoice_kind` は指定しない（指定するとAPIエラーになる）。

**摘要フォーマット例:**
```
カードA名称 引き落とし（2026年4月分）
```
`<name>` = `local.yml` の `credit_cards[].name`
「M月分」= 締め期間の月（費用が発生した月）

### 5.2 金額不一致の仕訳を修正

既存仕訳の ID を使って金額を更新する。

```
mfc_ca_putJournals
  id: <既存仕訳のID>
  amount: <期待引き落とし額>
```

---

## Step 6: 登録確認（MCP）

`mfc_ca_getJournals` で登録・修正した仕訳を取得し、内容を確認する。

### 最終レポート

```
引き落とし仕訳 完了レポート（カードA名称）:
  ✅ スキップ（登録済み・一致）: N 件
  🚀 自動登録: N 件
    - 2026-03-26: ¥X,XXX（仕訳No.XX）
  ✅ 修正: N 件
    - 2026-04-27: ¥X,XXX → ¥Y,YYY（仕訳No.XX）
  ⏳ スキップ（締め未完了）: N 件
    - 2026-06-26: 5月分（締め日 5/31 未到来）

すべての処理が完了しました。
```

---

## 仕訳ルール（確定）

未払金型（`type: business`）カードの課金（貸方: 未払金）に対応する引き落とし仕訳。カードごとに補助科目が異なる。
個人用（`type: personal`）は課金時に事業主借で直接記帳されるため本スキルの対象外。

| 借方勘定科目 | 借方補助科目 | 貸方勘定科目 | 貸方補助科目 | 摘要 |
|---|---|---|---|---|
| 未払金 | カード補助科目 | 普通預金 | 引き落とし口座の補助科目 | `<カード名> 引き落とし（YYYY年M月分）` |

- カード補助科目・引き落とし口座の補助科目・カード名の実値、締め日・引き落とし日のサイクルは
  `local.yml` の `credit_cards:` を参照（`name` / `closing` / `payment_day`）。勘定科目（未払金 / 普通預金）は
  `type: business` から導出。補助科目は連携時は自動仕訳から取得（`card_sub_account` / `bank_sub_account` は任意＝未連携手動登録時のみ）。
- 勘定科目 ID は Step 0.2 で MCP から動的に取得する。
- 摘要の「M月分」= 締め期間の月（費用発生月）。例: 5月26日引き落とし → 4月分。
- **営業日調整**：引き落とし指定日が土曜・日曜・日本の祝日の場合は翌営業日に繰り越し（例: 2026-04-26（日）→ 2026-04-27（月））。

---

## エラーハンドリング

| 状況 | 対応 |
|------|------|
| MCP 認証切れ | `mfc_ca_authorize` → `mfc_ca_exchange` を実施してリトライ |
| 期待額が 0 円 | 対象期間の費用仕訳なし → スキップしてレポートに記載 |
| 費用仕訳が想定より多い/少ない | 金額を表示してユーザーに確認を求めてから登録 |
| 引き落とし日が未来 | レポートに「未到来」として表示、登録はしない |

## セキュリティ注意事項

- 未登録の自動登録は締め完了分のみ。締め未完了分は絶対に登録しない
- 金額不一致の修正は必ずユーザー確認後に実行する

共通注意事項は `.claude/skills/_shared/security.md` を参照。
